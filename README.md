# QMS Frontend - Vue 3 Messenger Application

Браузерный фронтенд для QMS (Quantum Messaging System) - высокофункционального мессенджера на Vue 3 с поддержкой видео/аудио звонков через WebRTC и real-time обновлениями через WebSocket.

## 🚀 Возможности

### Аутентификация
- ✅ Регистрация новых пользователей
- ✅ Вход с двухфакторной аутентификацией (email подтверждение)
- ✅ Управление сеансами/устройствами
- ✅ Безопасное хранение токенов

### Чаты
- ✅ Список всех чатов с сортировкой по последнему сообщению
- ✅ Поиск и создание новых чатов
- ✅ Приватные чаты с пользователями
- ✅ Групповые чаты
- ✅ Поиск пользователей по имени, email или UIN
- ✅ Отправка текстовых сообщений
- ✅ Загрузка и скачивание файлов

### Звонки (WebRTC)
- ✅ Аудио звонки
- ✅ Видео звонки
- ✅ Real-time обновление статуса звонка
- ✅ Управление микрофоном и камерой
- ✅ Таймер звонка

### Профиль
- ✅ Управление профилем пользователя
- ✅ Установка и смена username
- ✅ Управление статусом (online/away/busy/offline)
- ✅ Выбор языка локали
- ✅ Просмотр активных сеансов
- ✅ Завершение удаленных сеансов

### Интеграция
- ✅ WebSocket для real-time сообщений
- ✅ WebRTC для видео/аудио звонков
- ✅ Axios с интерцепторами для API
- ✅ Pinia для управления состоянием
- ✅ Vue Router для навигации

## 📋 Требования

- Node.js: ^20.19.0 или >=22.12.0
- npm: 10.x или выше
- API сервер QMS (Laravel): работающий на http://localhost:8000
- Docker (опционально, для контейнеризации)

## 🛠️ Установка

### Локальная разработка

1. Клонируйте репозиторий:
```bash
cd /Users/andrei/projects/qms-frontend
```

2. Установите зависимости:
```bash
npm install --legacy-peer-deps
```

3. Создайте файл `.env.local` на основе `.env.example`:
```bash
cp .env.example .env.local
```

4. Отредактируйте переменные окружения в `.env.local`:
```env
VITE_API_URL=http://localhost:8000/api/v1
VITE_WS_URL=http://localhost:8000
VITE_API_TIMEOUT=10000
```

5. Запустите dev сервер:
```bash
npm run dev
```

Приложение будет доступно по адресу: http://localhost:5173

### Docker

1. Убедитесь, что API сервер запущен (порт 8000)

2. Запустите Docker Compose:
```bash
docker-compose up
```

Приложение будет доступно по адресу: http://localhost:3000

**Примечание:** В docker-compose.yml используется внутренняя сеть `qms-network`. Убедитесь, что API сервер также подключен к этой сети.

## 🔌 API Интеграция

Приложение взаимодействует с QMS API (Laravel) через:

1. **REST API** (Axios) для:
   - Аутентификации
   - Управления профилем
   - Операций с чатами и сообщениями
   - Управления вызовами

2. **WebSocket** (Socket.io) для:
   - Real-time получения сообщений
   - Обновлений статуса чатов
   - Уведомлений о входящих звонках
   - Синхронизации пользователей

3. **WebRTC** (Simple Peer) для:
   - Аудио звонков
   - Видео звонков
   - Обмена медиапотоками

## 🎯 Переменные окружения

| Переменная | Значение по умолчанию | Описание |
|------------|----------------------|---------|
| `VITE_API_URL` | `http://localhost:8000/api/v1` | URL базы API |
| `VITE_WS_URL` | `http://localhost:8000` | WebSocket URL |
| `VITE_API_TIMEOUT` | `10000` | Таймаут API запросов (мс) |
| `VITE_WS_RECONNECTION_ATTEMPTS` | `5` | Попытки переподключения WebSocket |
| `VITE_WS_RECONNECTION_DELAY` | `1000` | Задержка переподключения WebSocket (мс) |

## 🚦 Статусы звонков

| Статус | Описание |
|--------|---------|
| `ringing` | Звонок идет (ожидание ответа) |
| `active` | Звонок активен |
| `ended` | Звонок завершен |
| `rejected` | Звонок отклонен |
